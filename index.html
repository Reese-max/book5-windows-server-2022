part 01 · 安全基礎 — 物件描述
09 / 45
Security Descriptor

每個 物件 都帶著三張身分證

檔案、資料夾、印表機、登錄機碼…… 所有 Windows 物件都附有 Security Descriptor，描述它的擁有者、誰能存取、哪些動作要稽核。

01 · OWNER

擁有者

記錄物件的擁有 SID。擁有者預設能修改自己的 DACL，是物件的最終裁量者。

02 · DACL

Discretionary ACL

由一條條 ACE 組成，定義哪些 SID 可以做哪些動作——讀、寫、執行、刪除。

03 · SACL

System ACL

稽核用 ACL。指定哪些 SID 對物件的哪些操作需要被寫入安全事件日誌。

part 01 · 存取 vs 稽核
10 / 45
DACL vs SACL

一個管 能不能做，一個管 有沒有被記

DACL

誰可以動我？

  • 定義 SID 的允許 / 拒絕清單
  • 缺少 ACE = 預設拒絕
  • 顯式拒絕優先於允許
  • 系統存取前必查 DACL
vs
SACL

誰動了我？

  • 定義要稽核的 SID 與動作
  • 寫入 Security Event Log
  • 支援成功 / 失敗分別記錄
  • 需配合稽核原則才生效

part 02 · Share Permission
16 / 45
Share Permission

分享入口的 三層門

Share 權限只在「網路存取」這條路徑生效，本機登入時不會被檢查。 它是一道粗顆粒度的閘門。

LEVEL 1

Read

檢視檔案、執行程式、瀏覽子資料夾。最常見的最小授權。

LEVEL 2

Change

讀取 + 建立、修改、刪除檔案與資料夾。等同一般使用者操作。

LEVEL 3

Full Control

Change + 變更權限與接管擁有權。除管理員外應極力避免授予。

part 02 · NTFS Permission
17 / 45
NTFS Permission

更細顆粒度，本機與網路都吃

FC
Full Control

所有動作 + 改權限 + 接管擁有權。

MD
Modify

讀寫、刪除自己建立的檔案，不能改權限。

RX
Read & Execute

讀取並可執行檔案，最常見的開放等級。

LS
List Folder

僅瀏覽目錄結構，不開啟內容。

RD
Read

讀取檔案內容與屬性。

WR
Write

建立新檔、附加內容，不刪不執行。

part 02 · 最佳實務
23 / 45
best practice

Share 放寬，NTFS 細控

  1. 01
    Share 設為 Authenticated Users · Full Control

    避免在分享層做精細管控，否則維護成本高且容易踩雷。

  2. 02
    NTFS 採「群組授權，使用者繼承」

    建立角色群組（Finance / HR / R&D），把人放進群組，不要直接授權給個人。

  3. 03
    最小權限原則（Least Privilege）

    每個群組只給「剛好夠用」的權限。Full Control 留給管理員。

  4. 04
    避免使用 Everyone

    用 Authenticated Users 取代，至少要求過身分驗證才能進入。

part 04 · UAC

User Account Control

權限提升的 最後一道確認

即使是 Administrator 帳號，平時也以標準 token 運行。 只有當動作需要系統管理員權限時，才彈出 UAC 提示——這是「以最小權限為預設」的設計。

step 01

Split Token

登入時建立標準 + 管理員兩個 token，預設用標準的。

step 02

Elevate Prompt

偵測到需要提升的動作（如改系統檔），切換到安全桌面詢問。

step 03

Audit Trail

提升行為寫入事件日誌，未來可追蹤。

part 04 · 防火牆
36 / 45
Defender Firewall

三種網路情境，三套規則

Domain Profile

網域

主機加入 AD 網域且偵測到 DC 時自動套用。最寬鬆，但仍需精挑細選。

Private Profile

私人

家用 / 辦公室可信網路。可信任度中等，仍應守住關鍵服務。

Public Profile

公用

機場、咖啡店等不可信網路。預設全擋，僅放行必要連線。

原則：不要把防火牆關掉， 要學會針對情境細控，這是 owner 應有的顆粒度意識。
